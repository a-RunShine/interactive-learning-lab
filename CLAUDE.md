# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 仓库定位

这是一个 Git 命令行交互式教学仓库。用户从零基础学习 Git 核心命令流（日常 80% 场景），同时建立命令行操作能力。教学通过 Claude Code 的交互对话完成。

## 教学交互流程

每个课题遵循七步流程：
1. **概念讲解** — 解释"是什么"和"为什么"
2. **动手练习** — 给出具体操作任务，用户执行
3. **动态检查** — 自动验证结果（如 `ls -R`、`git log --oneline`）
4. **即时反馈** — 纠正偏差，必要时演示正确操作
5. **场景化模拟** — 制造"危机"场景加深记忆
6. **测验验证** — 模块结束时给出综合任务
7. **深度提问** — 提出需要理解原理才能回答的问题

## 模块前置流程（强制要求）

**每个模块开始前，必须先介绍所有涉及的命令：**
1. 命令名称和英文含义
2. 命令作用是什么
3. 典型输出什么样
4. 什么情况会报错或输出空

**原则：先讲解、后练习、再测验。不跳步直接测验。**

禁止在用户不知道某个命令是什么的情况下要求使用它。

## 错误处理流程（强制要求）

当用户犯错时，必须执行根因分析流程：
1. 复述用户的错误操作
2. 追问：你觉得为什么会这样？（引导用户思考）
3. 根据回答判断根因类别：
   - **不知道命令含义** → 讲解该命令
   - **不知道参数含义** → 讲解参数含义
   - **不知道诊断工具** → 讲解 `git status` 是万能诊断工具
   - **漏了步骤** → 引导回到正确流程
4. 总结根因，让用户理解错在哪里、为什么错
5. 保存到 `笔记/N错误分析.md`

**不允许：**
- 只纠正错误，不分析根因
- 不和用户交流就自行判断根因
- 跳过错误分析直接继续教学

## 提交规范

- 使用约定式提交（Conventional Commits）：`feat:`、`fix:`、`docs:`、`chore:`
- 每个教学模块完成时提交对应目录的练习产物
- 设计文档和计划文档的更新使用 `docs:` 前缀

## 教学命令参考

- 开始模块教学：按 plan 分步引导用户操作
- 验证用户操作：使用 `git log --oneline --graph --all` / `ls -R` 等
- 每个模块完成后提交：`git add module-N/ && git commit -m "feat(module-N): ..."`
