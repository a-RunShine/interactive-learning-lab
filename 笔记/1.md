# 模块 1：Git 起步 — 知识点总结

---

## 1. Git 核心概念

**版本控制解决的问题：**
- 记录每次修改（改了什么、什么时候、谁改的）
- 随时回溯到任意历史版本
- 多人协作时不打架

**Git vs GitHub：**
- **Git** = 本地版本控制工具
- **GitHub** = 托管 Git 仓库的云平台

---

## 2. Git 三棵树模型（最重要）

```
工作区 (Working Directory)     ← 你正在编辑的文件
       ↓ git add
暂存区 (Staging Area / Index)  ← 准备提交的内容
       ↓ git commit
仓库 (.git/)                   ← 历史记录
```

| 区域 | 说明 |
|------|------|
| 工作区 | 你的文件夹，正在编辑的内容 |
| 暂存区 | 临时存放，准备"拍照"的内容 |
| 仓库 | 真正的历史记录快照 |

**为什么要分三层？** — 让你审阅自己想提交什么，可以只提交部分改动。

---

## 3. 文件状态流转

```
未跟踪 (Untracked)  ──git add──→  已暂存 (Staged)
                                      │
                                      ↓ git commit
                                  已提交 (Committed)
                                      │
                                      ↓ 修改文件
                                  已修改 (Modified) ──git add──→ 已暂存
```

四种状态：**未跟踪、已修改、已暂存、已提交**

---

## 4. 核心命令详解

### `git init`
初始化仓库，让 Git 开始监控这个目录。

```bash
git init
# 输出: Initialized empty Git repository in .../.git/
```

### `git config`
配置用户信息（仅对当前仓库有效）

```bash
git config user.name "DongBo He"
git config user.email "your@email.com"
```

### `git status`
查看当前状态
- 红色 = 工作区有改动 / 未跟踪
- 绿色 = 已到暂存区

```
On branch master
Changes not staged for commit:
    modified: README.md       ← 红色
Changes to be committed:
    new file: temp.txt       ← 绿色
```

### `git add`
把文件从工作区移到暂存区

```bash
git add <文件名>     # 添加单个文件
git add .            # 添加所有文件（慎用）
```

### `git commit`
把暂存区内容保存为历史快照

```bash
git commit -m "提交信息"
```

**提交信息规范（约定式提交）：**
- `feat:` — 新功能
- `fix:` — 修复
- `docs:` — 文档
- `chore:` — 杂事

### `git log`
查看提交历史

```bash
git log --oneline    # 简洁模式
```

输出示例：
```
e94f230 (HEAD -> master) docs: update progress
1f86cac docs: init project
```

### `git diff`
查看工作区的改动（未暂存的内容）

```bash
git diff              # 工作区 vs 暂存区
git diff --staged     # 暂存区 vs 上次提交
```

输出格式：
```diff
- 删除的内容
+ 新增的内容
```

### `git restore`
撤销改动（Git 2.23+ 引入的新命令）

```bash
git restore <文件>              # 撤销工作区改动
git restore --staged <文件>     # 撤销暂存区（移回工作区）
```

---

## 5. HEAD 指针

**`HEAD` = 指针，指向当前所在位置（当前提交）**

```
e94f230 (HEAD -> master) docs: update progress
1f86cac docs: init project
```

- `HEAD` → `master` → 当前分支的最新提交
- `HEAD` 永远指向**当前分支**的最新提交

---

## 6. 工作流记忆图

```
┌─────────────────────────────────────────────┐
│  你写代码 (.md, .txt...)                    │
│         ↓ 编辑                              │
│  工作区 (Working Directory)                  │
│         ↓ git add                           │
│  暂存区 (Staging Area)                      │
│         ↓ git commit                        │
│  本地仓库 (.git/)                           │
│         ↓ git push                          │
│  远程仓库 (GitHub)                          │
└─────────────────────────────────────────────┘
```

---

## 7. 常见问题

| 问题 | 原因 | 解决方法 |
|------|------|---------|
| `LF will be replaced by CRLF` | Windows 换行符差异 | 正常，不影响功能 |
| `nothing to commit` | 工作区没有改动 | 确认文件是否修改 |
| `fatal: you must specify path(s)` | 没给文件名参数 | 加文件名 |

---

## 8. 关键区分

| 命令 | 作用范围 | 比喻 |
|------|---------|------|
| `git commit` | 仅本地 `.git/` | 写日记保存到本地 |
| `git push` | 上传到 GitHub | 把日记同步到云端 |

> **注意**：`git commit` 只是在本地保存，GitHub 不会收到任何通知！
