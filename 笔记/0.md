### 三个最基础的命令，你每分钟都会用到：

pwd — Print Working Directory（打印当前目录）

你此刻在哪？敲 pwd 就知道了。

ls — List（列出目录内容）

常用变体：

ls -l — 详细列表（权限、大小、修改时间）
ls -a — 显示隐藏文件（以 . 开头的文件）
ls -R — 递归显示子目录

cd — Change Directory（切换目录）

cd module-0 — 进入 module-0 目录
cd .. — 回到上级目录（.. 就是"上级目录"的意思）
cd . — 留在当前目录（. 就是"当前目录"的意思）
cd ~ — 回到你的家目录
cd /c/Users/k/Desktop/WorkSpace-CC/git学习 — 用绝对路径

### 绝对路径 vs 相对路径：

绝对路径：从根目录开始写全，如 /c/Users/k/Desktop
相对路径：从当前位置开始推算，如 cd module-0（假设你在它的父目录里）

echo 的字面意思是"回声"——把你给它的内容原封不动地打印出来。


echo hello
输出：hello


echo "我今天开始学 Git"
输出：我今天开始学 Git

就这么简单，它就是个"复读机"。

那为什么创建文件要用它？

关键在 > 这个符号。它是重定向操作符，意思是"把输出从屏幕转到文件里去"：


echo "hello"        → 在屏幕显示 hello
echo "hello" > a.txt → 不在屏幕显示，而是写入 a.txt
你还可以用 cat 验证：


echo "hello" > a.txt
cat a.txt           → 输出 hello
类似的还有 >>（追加，不覆盖）：


echo "line1" > a.txt    # a.txt 内容：line1
echo "line2" >> a.txt   # a.txt 内容：line1\nline2（追加）
echo "line3" > a.txt    # a.txt 内容：line3（覆盖了！）

### 退出困境

退出困境——Vim
这是你未来一定会遇到的情况。当你运行 git commit 但没有加 -m 参数时，Git 会打开一个叫 Vim 的编辑器，让你写提交信息。然后你会发现——不知道怎么退出去。

先让你体验一下：


vim practice.txt
你会进入一个看起来"卡住了"的界面。现在告诉你口诀：

三步退出法：

按 i — 进入"插入模式"（屏幕左下角会出现 -- INSERT --），现在可以打字
输入一些文字，比如 hello vim
按 Esc — 退出插入模式，回到"普通模式"
输入 :wq 然后回车 — 保存并退出（write and quit）
如果不想保存直接退出： :q! 然后回车（quit force）

你试一下，进 Vim 写点东西，然后用 :wq 退出。

###  命令行挑战
命令行技巧 + 盲打挑战
三个能显著提升你效率的技巧：

1. Tab 补全
不用完整输入文件名或路径，敲前几个字母按 Tab 自动补全：


cd practi[按 Tab]   自动补全为 cd practice.txt

2. ↑ 历史命令
按上箭头可以回看之前敲过的命令，不用重新打。

3. Ctrl+C 中断
如果你不小心启动了某个停不下来的命令，按 Ctrl+C 强制中断。

盲打挑战： 让我创建个长目录名，你的任务是只敲前 3-4 个字母，用 Tab 补全进入，不能完整输入目录名：

rm 文件名 → 删文件
rm -r 目录名 → 删目录
rm -rf 目录名 → 强制删目录（慎用，删了找不回来）

通配符 * 是命令行的大杀器：


touch a.java b.java c.class d.class   # 创建 4 个文件
ls *.java                              # 只看 .java 文件 → a.java b.java
ls *.class                             # 只看 .class 文件 → c.class d.class
rm *.class                             # 删除所有 .class 文件，不用一个个删
ls                                     # 确认只剩 .java 文件
