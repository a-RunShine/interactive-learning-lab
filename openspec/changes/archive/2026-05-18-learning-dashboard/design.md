## Context

当前教学体系缺少学习历史的持久化记录。用户完成教学会话后无法回顾学过的模块和犯过的错误。需要一个不下线就能打开的本地仪表盘。

## Goals / Non-Goals

**Goals:**
- 用户可离线打开 `dashboard.html` 查看学习历史
- 展示每次教学会话的主题、日期、模块完成情况
- 展示记录的错误日志（错误信息、根因、修复方案）
- skill 在教学结束时自动追加数据到 `dashboard-data.json`

**Non-Goals:**
- 不引入外部依赖（纯 HTML + CSS + JS）
- 不实现服务端存储或同步
- 不实现用户认证

## Decisions

| 决策 | 选项 | 选择 | 理由 |
|------|------|------|------|
| 数据存储 | JSON vs SQLite vs localStorage | **JSON 文件** | Agent 可直接读写，无需额外工具 |
| 前端框架 | React/Vue vs 纯 HTML | **纯 HTML** | 零依赖，git clone 即开 |
| 数据追加时机 | 每步 vs 每模块 vs 每次会话 | **每次会话结束** | 避免频繁写磁盘，数据粒度够用 |
| 图表库 | Chart.js vs 手写 CSS | **手写** | 数据量小，进度条足够 |

## Data Flow

```
教学会话完成后
  │
  ▼
Agent 读取 dashboard-data.json（如存在）
  │
  ▼
Agent 追加本次会话记录：
  ├── topic / date
  ├── modules[].name / completed
  └── errors[].error / rootCause / resolution
  │
  ▼
Agent 写回 dashboard-data.json
  │
  ▼
用户打开 dashboard.html
  │
  ▼
fetch('dashboard-data.json') → 渲染为卡片视图
```

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| dashboard-data.json 可能因用户拒绝写入而不存在 | 页面检测文件不存在时显示空状态引导 |
| JSON 文件在 git 中可能产生冲突 | 单文件，每个 agent 追加写入，冲突概率低 |
| 用户删除 dashboard-data.json 丢失历史 | 页面友好提示"还没有学习记录，开始一次教学吧" |
