## Context

当前 skill 执行 Phase 2 时未规定教学环境的创建策略。不同 AI 编码 Agent 的默认行为：
- OpenCode：倾向于纯对话，不写文件（权限严格）
- Claude Code：倾向于创建临时文件
- Codex：取决于配置

需要统一规定，确保教学体验一致。

## Goals / Non-Goals

**Goals:**
- 定义统一的"教学环境管理"协议
- 确保 agent 在教每个模块时创建专用目录
- 确保练习文件有组织地存放
- 教学完成后提供清理选项

**Non-Goals:**
- 不强制特定目录命名规则
- 不支持跨会话的持久化（练习文件由用户决定保留或删除）

## Decisions

| 决策 | 选项 | 选择 | 理由 |
|------|------|------|------|
| 目录位置 | 仓库根 vs 用户 home vs 临时目录 | **仓库根** | Agent 有仓库写权限，目录在项目中可见 |
| 目录命名 | `learn-<topic>/module-N-<name>/` | 用户可协商 | 清晰表明学习意图 |
| 文件清理 | 全留 vs 全删 vs 询问 | **询问** | 尊重用户选择 |
| 跨模块共享 | 共享 vs 独立目录 | **独立** | 模块间逻辑分离，避免混淆 |

## Architecture

```
进入 module-1「变量与类型」
  │
  ├── 创建 learn-python/module-1-types/
  │       ├── exercise_01.py      ← 步骤2 原子任务
  │       ├── buggy.py            ← 步骤5 危机模拟
  │       └── quiz.py             ← 步骤6 结课测验
  │
  └── 完成后 → 进入 module-2

全部模块完成后：
  "练习文件在 learn-python/ 目录下，要保留还是删除？"
```

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| 用户不想写文件（只想学理论） | 原子任务步骤中允许非操作型领域用问答替代 |
| 目录积累过多 | 每次进入新模块前检查是否存在同名目录，询问是否覆盖 |
| Agent 无写权限 | 退回到纯对话模式，提示"当前环境无法创建练习文件" |
