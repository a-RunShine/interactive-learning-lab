## ADDED Requirements

### Requirement: Dashboard SHALL merge data from dual sources
`dashboard.html` SHALL load both `active-topics.json` (pre-generated by Agent from `teach-state.py list`) and `dashboard-data.json`, merging on topic name.

#### Scenario: Topic present in both sources
- **WHEN** a topic has an active checkpoint (in active-topics.json) AND an archived session (dashboard-data.json)
- **THEN** the dashboard SHALL display module progress from active-topics.json, errors from dashboard-data.json, and domain from active-topics.json (or dashboard-data.json if active-topics.json omits it)

#### Scenario: Topic only in active checkpoint
- **WHEN** a topic has an active checkpoint in active-topics.json but no archived session
- **THEN** the dashboard SHALL display the topic with module progress from list and no errors

#### Scenario: Topic only in archived data
- **WHEN** a topic has an archived session in dashboard-data.json but no active checkpoint
- **THEN** the dashboard SHALL display the topic as completed, using archive data for module names and error records

### Requirement: Dashboard SHALL group learning topics by domain
`dashboard.html` SHALL group learning sessions by their `domain` field, displaying each domain as a section with all its topics.

#### Scenario: Multiple domains with topics
- **WHEN** sessions contain topics with domains "编程语言" and "工具与平台"
- **THEN** the dashboard SHALL render two separate domain sections, each with its own progress summary and topic cards

#### Scenario: Topic without domain
- **WHEN** a topic has no `domain` field
- **THEN** the dashboard SHALL group it under a default section e.g. "未分类"

### Requirement: Dashboard SHALL display learning streak
`dashboard.html` SHALL calculate and display the count of consecutive calendar days with learning activity.

#### Scenario: Active streak
- **WHEN** sessions have dates for today, yesterday, and the day before
- **THEN** the dashboard SHALL display "连续 3 天"

#### Scenario: Broken streak
- **WHEN** the most recent session is yesterday and there is no session today and two days ago
- **THEN** the dashboard SHALL display "连续 1 天" (yesterday only)

#### Scenario: No learning at all
- **WHEN** there are no sessions
- **THEN** the dashboard SHALL display "连续 0 天" or hide the streak indicator

### Requirement: Dashboard SHALL show learning timeline
`dashboard.html` SHALL render a chronological timeline of learning activities grouped by "今天"/"昨天"/date labels.

#### Scenario: Timeline with recent and past entries
- **WHEN** sessions exist for today, yesterday, and older dates
- **THEN** the timeline SHALL group entries under "今天", "昨天", and specific date labels for older entries

#### Scenario: Timeline shows module progress
- **WHEN** a timeline entry is rendered for an active topic
- **THEN** the entry SHALL show topic name, current module name, and current step number

#### Scenario: Timeline marks completed modules
- **WHEN** a timeline entry corresponds to a module that was completed
- **THEN** the entry SHALL visually indicate completion (e.g. checkmark)

### Requirement: Dashboard SHALL have domain-based filter tabs
`dashboard.html` SHALL provide filter tabs that allow the user to view topics from all domains or a specific domain.

#### Scenario: Filter by specific domain
- **WHEN** user clicks a domain filter tab (e.g. "编程语言")
- **THEN** only topics belonging to that domain SHALL be displayed in the domain sections and timeline

#### Scenario: Reset to all domains
- **WHEN** user clicks the "全部" filter tab
- **THEN** all topics from all domains SHALL be displayed

### Requirement: Dashboard SHALL maintain error log section
`dashboard.html` SHALL retain the existing error log section showing error entries with message, module, root cause, and resolution, grouped or filtered consistently with the current domain/topic view.

#### Scenario: Error log filtered to active domain
- **WHEN** user filters by a specific domain
- **THEN** the error log SHALL show only errors from sessions in that domain
