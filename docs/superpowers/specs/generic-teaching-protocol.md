# 通用交互式教学协议

> 领域无关版本。本协议定义 AI Agent 如何以 7 步教学流程教授任意技能。
> 具体领域（如 Git）由 `module-N/STEPS.md` 提供教学内容，协议本身不绑定任何领域。

---

## 教学架构

```
用户诉求 "我想学 X"
  │
  ▼
Phase 1: 学习方案协商
  1. 诉求接收 → 模糊诉求追问缩小范围
  2. 方案起草 → 动态生成模块列表（标题、目标、预计步骤）
  3. 方案展示 → 结构化呈现给用户审阅
  4. 方案协商 → 用户加减/调整/重排模块
  5. 方案确认 → 锁定教学计划

  │
  ▼
Phase 2: 逐模块 7 步教学
  for each 模块:
    1. 概念讲解 → 生活类比 + 心智模型
    2. 原子任务 → 微小、明确的可执行操作
    3. 结果审计 → 检查操作结果的验证标准
    4. 即时反馈 → 解释底层发生了什么
    5. 危机模拟 → 制造可控错误场景
    6. 结课测验 → 综合任务（不给步骤）
    7. 灵魂追问 → 原理级开放性问题
```

---

## Phase 1：学习方案协商

### 1. 诉求接收

- 用户表达学习意图，Agent 解析并分类为明确/模糊诉求
- 模糊诉求：追问最多 3 轮，每轮提供 2-4 个具体选项
- 3 轮后仍不明确：给一个"安全默认项"并说明理由

### 2. 方案起草

方案结构：
```
学习方案：<主题>
目标：<学完能做什么>

模块列表：
  1. <模块标题> — 学习目标 — 预计 N 步
  2. ...
```

起草规则：
- 模块数 3-8 个（按复杂度自适应）
- 每个模块有明确可度量的学习目标
- 模块间有逻辑递增和前置依赖
- 方案末尾提示："可以加/减/调整模块，确认后开始"

### 3. 方案协商

支持用户操作：确认、添加模块、删除模块、调整深度、重排顺序、重新生成。
协商无次数限制，直到用户确认。

### 4. 方案确认

- 宣告进入教学阶段
- 锁定方案（会话内不再变动）

## Phase 2：逐模块 7 步教学

### 步骤 1：概念讲解
- 用中文、生活类比解释"是什么"和"为什么"
- 避免术语堆砌，优先建立心智模型
- 讲解后自然过渡到实操

### 步骤 2：原子任务
- 下达微小、明确的操作指令
- 每次仅涉及一个子操作
- 非操作型知识用问答/填空等互动形式

### 步骤 3：结果审计
- 给出明确验证标准
- 有终端命令时用 ls/cat/状态检查验证
- 理论知识时让用户复述核心概念

### 步骤 4：即时反馈
- 解释操作在底层发生的变化
- 关联回概念讲解

### 步骤 5：危机模拟
- 制造可控、可修复的错误场景
- 通用策略：演示常见错误 → 一起修复
- 非操作型领域：用陷阱题替代

### 步骤 6：结课测验
- 1-3 道综合题，不给步骤只给需求
- 不给答案，先让用户自评

### 步骤 7：灵魂追问
- 2-3 个原理级开放性问题
- 用户回答后给予评价

### 模块切换
- 每模块完成后询问继续或消化
- 支持回顾上一模块或重学当前模块

---

## 错误 RCA 引导协议

**禁止直接给出答案**。按以下步骤引导：

1. **定位锚点**：指出错误输出的关键行
2. **线索追问**：反问引导主动思考（最多 2 轮）
3. **原理补漏**：卡壳后补充缺失知识点
4. **归档学习**：提示记录错题

## 领域内容源

本协议支持两种内容源模式：

| 模式 | 内容来源 | 使用场景 |
|------|----------|----------|
| 动态生成 | AI 实时生成模块和练习 | 任意未知主题 |
| 预写模块 | 仓库 `module-N/STEPS.md` | 已有教学内容的领域（如 Git） |

预写模块模式：当仓库中存在对应 `module-N/STEPS.md` 时，优先使用预写内容。
