# Git 命令行教学系统 — 设计文档

## 概述

通过 Claude Code 交互式教学，帮助用户从零掌握 Git 核心命令流（日常 80% 场景），同时建立命令行操作能力。教学形式：概念讲解 → 动手练习 → 测验验证 → 深度提问。

## 前提

- **用户起点**：零命令行基础、零 Git 基础
- **学习目标**：掌握核心 Git 命令流 + 命令行操作能力，为后续开发工作铺路
- **学习节奏**：混合模式，每个主题先讲概念，立刻动手练习，最后通过测验验证
- **深度选择**：核心命令流（A 方案），覆盖日常 80% 场景，不赶时间、扎实为主

## 课程架构

```
模块0: 命令行生存手册        ← 命令行的「第一扇门」
模块1: Git 起步              ← 版本控制的基本单元
模块2: 分支管理              ← Git 的核心超能力
模块3: 远程协作              ← 连接世界
模块4: 历史追溯与回退        ← 时间旅行
模块5: 综合实战              ← 模拟真实工作流
```

每个模块遵循：概念讲解 → 动手练习 → 测验验证 → 深度提问。
仓库内为每个模块创建独立练习目录（`module-0/` 至 `module-5/`）。

## 模块详情

### 模块 0 — 命令行生存手册
目标：建立命令行操作能力，为 Git 学习铺路。

- 认识终端：Shell、终端、命令行的概念，Windows 下的 Bash 环境
- 导航文件系统：`pwd`、`ls`、`cd`、绝对路径 vs 相对路径
- 文件操作：`mkdir`、`touch`、`rm`、`mv`、`echo`、`cat`
- 通配符：`*`、`?`、`[abc]`，实战 `rm *.class`、`ls *.java`
- 命令行技巧：Tab 补全、↑ 历史命令、Ctrl+C 中断
- 退出困境：Vim 简介，`i` 插入模式、`Esc` 退出、`:wq` 保存退出、`:q!` 强制退出；为什么 `git commit` 没加 `-m` 会进入 Vim

### 模块 1 — Git 起步
目标：理解版本控制的基本模型，掌握工作流核心三步。

- 为什么需要版本控制：手动备份的痛点
- Git 的三棵树：工作区 → 暂存区 → 仓库（核心心智模型）
- 第一次提交：`init` → `config` → `status` → `add` → `commit`
- 查看历史：`log` 及其常用选项
- .gitignore：忽略编译产物（`*.class`、`target/`）、IDE 配置（`.idea/`）、临时文件，glob 语法

### 模块 2 — 分支管理
目标：掌握 Git 最强大的特性——分支。

- 什么是分支：指针的概念
- 分支命名规范：`feat/xxx`、`fix/xxx`，语义化命名的职业习惯
- 创建与切换：`branch`、`switch`/`checkout`
- 合并分支：`merge`、fast-forward vs 三方合并
- 冲突解决：冲突是如何产生的、手动解决流程

### 模块 3 — 远程协作
目标：理解 Git 分布式本质，掌握协作命令。

- 分布式模型：本地仓库 vs 远程仓库、`clone` 的含义
- SSH vs HTTPS：两种协议差异、为什么 SSH 更省事；`ssh-keygen` 生成密钥对、公钥配置到 GitHub/Gitee
- 远程操作：`remote`、`push`、`fetch`、`pull`
- 模拟协作：用本地目录模拟远程仓库，体验 push/pull 流程

### 模块 4 — 历史追溯与回退
目标：能安全地查看历史和撤销操作。

- 差异查看：`diff`——工作区 vs 暂存区 vs 提交
- 历史浏览：`log --oneline --graph --all` 可视化历史
- 安全回退：`revert`（安全）vs `reset`（危险但有分寸）
- reflog 后悔药：操作日志的概念、`git reflog` 找回"丢失"的提交
- 临时保存：`stash` 暂存工作现场

### 模块 5 — 综合实战
目标：模拟真实工作流，串联全部知识。

模拟完整开发场景：接到需求 → 开分支 → 开发 → 提交 → 合并 → 发现 bug → 回退 → 修复 → 推送。

## 教学交互模式

每个课题执行标准流程：
1. **概念讲解**：用中文思维解释"是什么"和"为什么"
2. **动手练习**：在对应 module 目录下给出具体操作任务，用户执行
3. **即时反馈**：用户执行后检查输出，纠正理解偏差
4. **测验验证**：模块结束时给出综合任务，验证掌握程度
5. **深度提问**：提出需要理解原理才能回答的问题，用户作答后给予反馈

## 技能依赖

教学过程中可能需要使用的 Superpowers 技能：
- `superpowers:verification-before-completion`：每个模块测验后验证用户操作结果
- 按需使用，不在当前阶段预设全部
